## batch?

```
    일괄처리라는 뜻을 가지고 있습니다. 한마디로 대용량의 데이터를 한번에 처리하는 것을 의미합니다.
     조건 4가지
	1. 자동화 - 심각한 문제 해결을 제외한 모든 데이터는 사용자 개입 없이 실행 되어야한다.(scheduler를 이용하여 자동실행을 하기 때문에)
	2. 대용량 데이터 - 대량의 데이터를 GET,POST,계산하는 등의 행위가 이루어져 처리 할 수 있어야합니다.
	3. 데이터를 가져오는 도중 충돌,에러 없이 처리할 수 있어야합니다.
	4. 지정한 시간 안에 처리를 완료하거나 동시에 실행되는 다른 어플리케이션을 방해하지 않도록 수행되어야 합니다.
     간단한 사용방법
	1. baych를 실행하기위한 의존성 추가 (implementation 'org.springframework.boot:spring-boot-starter-quartz')
	2. 프로젝트 메인실행 클래스에 @EnableScheduling 어노테이션 추가
	3. batch 클래스에 @Component 어노테이션을 추가
	4. batch를 실행할 메소드에 @Scheduled(cron="* * * * * *")을 추가하여 지정 시간에 일괄처리 할 수 있도록 해줍니다.
	5. (cron="* * * * * *")는 Batch Scheduler의 표현식으로 시간을 지정할 수 있습니다.
```

## spring boot security

```
    애플리케이션의 보안 즉, 인증 or 인가를 담당하는 스프링의 하위 프레임 워크입니다.
     대부분의 웹,앱의 필수 요소라고 할 수 있습니다.
     security에 많이 사용되는 인증 방법으로는 session과 cookie가 있습니다.
	인증 : 사람,인물에 대한 보안
		한마디로 누가 라는것이 핵심 포인트입니다.
		예를 들어 보호된 리소스에 접근대상이 누구인지, 누가 리소스에 접근하는지, 누가 접근하냐에 따라서 작업을 수행해도 되는지를 말합니다.
	인가 : 행동, 권한에 대한 보안
		접근하는 특정인물에 대한 권한을 확인하는 것이 인가 입니다.
		특정인물이 어떠한 권한을 가지고 있는지 확인하는 과정을 의미합니다.
```

## AWS

```
1. Amazon Web Services의 약자로 클라우드의 한 종류 입니다.
2. 아마존의 클라우드 서비스입니다.
3. AWS를 사용하는 이유는 확장성,탄력성,높은 접근성
4. 대표적인 사용 서비스 : EC2, RDS, S3
   **EC2(Elastic Cloud Compute)**
    1) 컴퓨터이다. 가상의 공간(실제 아마존 데이터베이스)에 나만의 컴퓨터를 등록 하는 것입니다.
    2) 웹,앱을 개발하기 위한 코드를 작성하면 그 코드를 저장하여 동작하게 하는 컴퓨터로 사용하는것을 의미합니다.

    **RDS(Relational Database Service)**
    1) 관계형 데이터베이스를 서비스하는 기능입니다.(관계형 DB)

    **S3(Simple Storage Service)**
    1) 영상,이미지등 파일을 저장해두는 기능을 제공하는 서비스입니다.
    2) 많은 양을 오래동안 안전하게 보관하여 사용할 수 있습니다.
```
